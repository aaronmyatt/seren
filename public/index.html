FCIS Browser Demo

Open your browser's Developer Console (F12 or Cmd+Option+J) to see the output.

The same pure Core functions that run on the JVM are running here in JavaScript. Only the Adapter layer differs — file-based storage on JVM, atom-based in-memory storage here.

What's happening

Core (pure, .cljc)     → validates & creates users
Adapter (.cljs)        → stores users in an atom (in-memory)
App (.cljc)            → wires Core + Adapter together

Try it yourself

In the console, you can call functions directly:

// Register a user through the full stack
fcis.app.main.register_user_BANG_({
  "store-dir": "demo",
  name: "Charlie",
  email: "charlie@example.com"
})

// List all users
fcis.app.main.list_all_users({"store-dir": "demo"})

// Pure validation (Core layer)
fcis.core.user.validate_email("test@example.com")
